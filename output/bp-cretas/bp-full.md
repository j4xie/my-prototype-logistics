## 2. 痛点与机遇 (Problem)

### 2.1 行业困境

中国食品加工行业正面临结构性压力升级：

| 痛点 | 数据支撑 |
|------|----------|
| 成本波动大、损耗高 | 我国食物损耗浪费率 **22.7%** (农业农村部食物与营养发展研究所, 2023)，折合全行业年损失超 **1.2 万亿元** |
| 监管趋严 | 2025 年 6 月七部委联合发布《食品工业数字化转型实施方案》，要求 2027 年重点企业业务管理数字化率达 **80%**，关键工序数控化率达 **75%** |
| 数据孤岛严重 | **55.6%** 企业认为数据孤岛导致决策延迟 (艾瑞咨询, 2025) |
| 管理依赖经验 | 仅 **3.2%** 企业实现智能驱动 (联想×36氪《中国中小企业数字化转型报告》, 2024) |
| 生产/质量/追溯仍是黑箱 | **62.6%** 中小企业转型仍处早期 (联想×36氪, 2024) |

### 2.2 现有方案的结构性失败

市场解决方案普遍陷入两极分化：

| 痛点 | 表现 | 代表 |
|------|------|------|
| **"用不起"** | 平均成本 50-100 万元，不含订阅与定制；实施 6-12 个月 | SAP、Oracle、金蝶、用友 |
| **"不适用"** | 通用型软件，缺乏食品行业深度理解；一线不愿用，沦为摆设 | 通用 MES/ERP |
| **"连不通"** | 单一环节孤岛，无法实现业务打通与管理统一视图 | 各类细分工具 |

### 2.3 为什么是现在？

- **政策窗口**: 2025 年 6 月工信部等七部委发布《食品工业数字化转型实施方案》，明确 2027 年重点企业数字化率 80%、2030 年全面普及，30-40 万家企业必须上系统
- **技术成熟**: 云计算+IoT+大语言模型的成本已降至中小企业可负担水平
- **制造+AI 市场爆发**: 从 2024 年的 60 亿美元，向 2032 年的 620 亿美元跃升 (CAGR ~34%)

## 3. 解决方案 (Solution)

### 3.1 核心理念

> **白垩纪是食品企业的 AI Agent — 不是让企业学会用软件，而是让 AI 学会管企业。**

传统 ERP/MES 要求工人适应系统；白垩纪反过来——用户用自然语言说"帮我查本月牛肉采购成本"，AI Agent 自动理解意图 → 调用数据 → 分析根因 → 返回可执行建议。这不是又一个 ERP，而是一个**能自主执行任务的食品工业 AI Agent 平台**。

### 3.2 产品架构: AI Agent + 五大能力层

| 层级 | 定位 | 核心价值 |
|------|------|----------|
| **AI Agent 核心** | 食品工业智能决策引擎 | 自然语言交互 → 意图识别 (98%+) → 自主执行任务 → 返回分析与建议 |
| **数据采集层** | IoT 硬件 + 移动端录入 | 自研硬件自动采集 + 扫码录入，确保数据真实可信 |
| **业务管理层** | 极简 SaaS 管理平台 | 生产管理、质量控制、仓储物流、采购销售、财务，**以及餐饮运营 (配方/领料/盘点/损耗)**——移动端零培训 |
| **分析洞察层** | SmartBI + LLM 分析 | Excel→图表→AI 洞察 30 秒；成本根因、异常预警、行业基准对比 |
| **连接生态层** | 上下游协同 | 品质码、供应链透明、消费者触达 (追溯是功能之一) |

### 3.5 AI Agent 平台深度解析

白垩纪的核心不是"管理系统"，而是一个**食品工业 AI Agent**——它能理解用户的自然语言指令，自主调用数据、执行分析、生成报告、触发流程。追溯、质检、库存管理等都是 Agent 能力的具体应用场景。

#### 3.5.1 多角色移动端 (React Native App)

针对食品工厂不同岗位设计了 6 种专属角色界面：

| 角色 | 核心功能 | 使用场景 |
|------|----------|----------|
| **工厂管理员** | 全局仪表盘、人员管理、系统配置、AI 报告 | 办公室/远程 |
| **车间主管** | 生产计划、排产调度、进度跟踪、产线监控 | 车间现场 |
| **质检员** | 检验记录、不合格处理、HACCP 检查点签核 | 质检岗位 |
| **仓储主管** | 入库/出库/盘点、库存预警、批次追溯 | 仓库 |
| **调度员** | 排产计划、设备调度、工单分配 | 调度中心 |
| **HR 管理员** | 考勤、白名单、部门管理 | 人事 |

**设计理念**: "零培训上手"——每个角色只看到自己需要的功能，扫码即可完成数据录入，无需理解整个系统。

#### 3.5.2 AI 智能分析引擎 (SmartBI)

这是白垩纪的核心差异化模块——**让中小工厂老板也能拥有"数据分析师"**。

**工作流程**:
1. **数据输入**: 上传 Excel/CSV 报表，或系统自动汇总生产数据
2. **智能解析**: Python 服务自动识别数据结构、检测行业类型 (食品加工/餐饮)
3. **图表生成**: 根据数据特征自动推荐最佳图表类型 (趋势图/饼图/热力图/瀑布图等)
4. **AI 洞察**: LLM 分析数据异常、成本结构、行业基准对比，生成自然语言报告
5. **交互问答**: 用户可以用自然语言追问 "为什么本月成本上升了？"，AI 会基于数据给出根因分析

**技术架构**:
- **SSE 流式传输**: 上传后实时推送解析进度、图表生成、AI 分析，用户体验流畅
- **行业自适应**: 自动检测食品子行业 (火锅/烘焙/预制菜等)，注入行业基准数据 (食材成本率、翻台率等)
- **多维度分析**: 支持跨 Sheet 分析、同比/环比对比、预测建模、相关性分析
- **端到端 30 秒**: 从 Excel 上传到完整分析报告，全程 30 秒

#### 3.5.3 AI Agent 核心: 意图识别与任务执行

这是白垩纪 AI Agent 的"大脑"——**混合意图识别 + 自主任务执行系统**。用户用自然语言下达指令，Agent 自动理解 → 路由 → 执行 → 返回结果：

| 层级 | 技术 | 作用 |
|------|------|------|
| 第一层 | BERT 分类器 (fine-tuned, 259 类) | 快速精准分类常见操作意图 |
| 第二层 | 语义关键词匹配 | 兜底层，处理关键词明确的查询 |
| 第三层 | LLM Function Calling (通义千问) | 处理复杂、模糊的自然语言请求 |

**实际效果**: 用户说"帮我查一下本月牛肉的采购成本"，系统自动识别意图 → 调用对应数据查询 → 格式化返回结果。准确率 **98%+** (1,232 条测试用例)。

#### 3.5.4 Web 管理后台 (Vue Admin)

面向工厂管理层和平台管理员的全功能 Web 端：

- **生产管理**: 生产计划、批次管理、BOM 配方、工序转换率
- **质量管理**: 检验标准、检验记录、不合格品处置、质量趋势
- **仓储物流**: 库存盘点、物料管理、出入库、发货管理
- **采购销售**: 供应商管理、采购订单、销售订单、客户管理
- **财务模块**: 成本核算、应收应付、财务报表
- **人力资源**: 员工管理、考勤、部门、设备排班
- **SmartBI 仪表盘**: 数据上传、可视化图表、AI 问答、分享功能
- **系统配置**: 用户权限、AI 意图管理、产品管理、设备管理

#### 3.5.5 餐饮运营模块 — 同一 AI Agent，第二赛道

同一套 AI Agent 引擎，无需额外开发即可服务连锁餐饮场景。这是平台可扩展性的最佳证明：**一个 AI Agent，两个垂直行业**。

**日常运营管理** (App + Web 双端覆盖):

| 功能 | 场景 | 覆盖端 |
|------|------|--------|
| 配方管理 | 标准化菜品配方、成本核算、毛利测算 | App + Web |
| 领料管理 | 按配方自动生成领料单、出库关联 | App + Web |
| 盘点管理 | 食材定期盘点、损耗对比、差异分析 | App + Web |
| 损耗追踪 | 废弃/过期/异常损耗登记、趋势监控 | App + Web |

**SmartBI 餐饮分析** (自动检测餐饮数据、注入行业基准):

| 分析维度 | 说明 |
|----------|------|
| 菜品四象限 | 销量 × 毛利矩阵，识别明星菜/问题菜/现金牛/瘦狗 |
| 门店对比 | 多门店营收、客单价、翻台率横向对比 |
| 大众点评差距 | 口味/环境/服务维度雷达图，对标竞争门店 |
| 品类结构 | 品类销售占比、趋势变化、SKU 贡献度 |

**AI Agent 意图示例** (18 类餐饮专属意图):

| 类别 | 示例查询 |
|------|----------|
| 菜品查询 | "帮我看下哪些菜毛利低于 50%"、"本月新品销量排名" |
| 食材管理 | "今天需要领多少斤牛肉"、"上周食材损耗率多少" |
| 营收分析 | "本月各门店营收对比"、"午市和晚市客单价差多少" |
| 竞争分析 | "和隔壁火锅店的点评评分差距在哪" |

> **关键洞察**: 餐饮模块验证了白垩纪的平台架构——同一 AI Agent 引擎、同一 SmartBI 分析框架、同一移动端交互模式，只需注入行业知识即可覆盖新的垂直行业。竞品只能做其一，白垩纪覆盖全链。

3. **行业协会渠道**: 通过食品行业协会、政府食品安全监管渠道批量获客
4. **需求驱动循环**: 下游品牌方追求供应链透明时，系统成为上游工厂的合作前提

## 5. 商业模式 (Business Model)

### 5.1 收入模型: Result-as-a-Service

核心理念: **按企业实际节省或创造的价值定价**，而非按功能模块收费。

| 收入来源 | 模式 | 阶段 |
|----------|------|------|
| SaaS 订阅 | 分层订阅 (基础版/专业版/企业版) | Phase 1-3 |
| AI 增值服务 | AI 分析报告、预测预警、定制模型 | Phase 2-3 |
| 硬件销售 | 自研 IoT 设备 (数据采集终端) | Phase 1-3 |
| 平台抽佣 | 上下游交易撮合、服务费 | Phase 3 |
| 标准授权 | 证书/认证/标准授权试运行 | Phase 3 |

### 5.2 定价策略

| 层级 | 月费 (预估) | 包含 | 目标客户 |
|------|-------------|------|----------|
| 基础版 | ~1,000 元/月 | 生产管理 + 基础 AI 问答 + 报表 | 微型工厂 (<50 人) |
| 专业版 | ~3,000 元/月 | + SmartBI 深度分析 + AI Agent 全功能 + 多角色 | 中小工厂 (50-200 人) |
| 企业版 | 定制报价 | + 硬件套装 + 定制集成 + 专属顾问 | 大型工厂 (200+人) |

**对比传统方案**: 传统 ERP/MES 年费 50-100 万元 → 白垩纪年费 **1.2-3.6 万元**，仅为 **1/14 到 1/28**。

**餐饮门店定价**: ~500-1,000 元/门店/月，连锁品牌按门店数阶梯定价。

### 7.1 竞争格局

### 7.2 核心竞品对比

| 维度 | **白垩纪** | 黑湖智造 | 客如云/哗啦啦 | 爱创科技 | 金蝶/用友 |

| 中小企业 (<50人) | **是** | 部分 (小工单) | 是 | 否 | 否 |
| 移动端优先 | **是** | 部分 | 是 | 否 | 否 |
| 零培训上手 | **是** | 否 | 部分 | 否 | 否 |
| 生产管理 (MES) | **是** | 是 | 否 | 否 | 是 |
| 餐饮运营 | **是** | 否 | 是 | 否 | 否 |

| 部署时间 | **小时级** | 天-周 | 小时 | 周 | 月 |

### 7.3 差异化壁垒

1. **AI Agent 架构**: 不是"加了个 AI 按钮"的管理系统，而是以 AI Agent 为核心的全新范式 — 自然语言驱动、自主任务执行、98%+ 意图识别
2. **食品行业深度**: 从第一行代码就为食品加工设计 — 质检流程、HACCP 合规、批次管理、行业基准全部内建
3. **极简移动端**: "零培训"设计理念，一线工人扫码即用；黑湖智造、金蝶等仍需 PC 端+培训
4. **软硬一体**: 自研 IoT 采集设备确保数据真实可信，不依赖手工录入
5. **Result-as-a-Service**: 按效果定价，大幅降低客户决策门槛
6. **数据飞轮**: 更多工厂使用 → 更多食品行业数据沉淀 → AI 模型更精准 (成本预测、异常检测) → 更高客户价值 → 更多工厂加入。通用制造 SaaS 的数据无法直接迁移到食品行业特有的批次追溯、HACCP 控制点、保质期管理等场景
7. **双赛道验证**: 同一 AI Agent 已成功服务工厂+餐饮两个垂直行业，证明平台可扩展性。竞品 (黑湖智造/客如云) 只能做其一，白垩纪覆盖全链

### 7.4 最大竞争威胁

**黑湖智造"小工单"** — 1 万元/年，面向首次数字化的中小工厂。但它是通用制造业产品，没有 AI Agent、没有食品行业智能分析、没有硬件。如果黑湖推出食品垂直版本，将是最直接的威胁。

**应对**: 白垩纪的 AI Agent 架构是根本性差异——黑湖做的是"数字化记录工具"，我们做的是"能自主分析和建议的 AI 顾问"。AI Agent 能力需要行业数据积累，先发优势一旦建立难以追赶。
