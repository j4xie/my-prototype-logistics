# 并行工作分析规则

**最后更新**: 2026-01-22

---

制定计划时，在末尾添加「并行工作建议」：

## Subagent 并行（单Chat内）

| 适合 | 不适合 |
|------|--------|
| 独立模块修改 | 严格依赖顺序 |
| 探索不同代码区域 | 需前一步结果 |
| 同时测试和审查 | 需人工确认 |

## 多Chat窗口并行

| 适合 | 不适合 |
|------|--------|
| 前后端独立开发 | 修改相同文件 |
| 不同模块互不影响 | 有git冲突风险 |
| 文档+代码 | 需共享上下文 |

## 输出格式

```markdown
## 并行工作建议
### Subagent: ✅/❌ [说明]
### 多Chat: ✅/❌ [说明] [冲突风险]
```
