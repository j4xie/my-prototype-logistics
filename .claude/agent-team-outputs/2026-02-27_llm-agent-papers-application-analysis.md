## 一、执行摘要 (Executive Summary)

四篇论文（GenEnv、DreamGym、AutoForge、AgentFrontier）围绕同一核心主题：**在能力边界处进行高密度训练能最大化学习效率**（即维果茨基"最近发展区"原理）。白垩纪系统已具备部分基础设施——三级语义路由（阈值0.88/0.70）、AtomicLong路由统计、PyTorch分类器（~185标签）、以及存在但从未在生产环境运行的增量微调脚本。然而，将RL Agent训练方法迁移到supervised NLU分类任务存在**根本性方法论落差**：RL有环境自动反馈，而NLU分类器的"能力边界"样本需要人工标注，系统当前完全缺乏用户反馈闭环。论文思想可提炼出4项可行改进（ZPD闭环、弱类合成、课程采样、工具文档测试），但预期收益需大幅下调——总准确率提升空间受限于65%流量走PHRASE_MATCH规则路径，ML模型改进被稀释。**建议优先投资用户反馈机制和模糊标签清理，而非直接套用论文方法。**

## 二、论文概览

### 论文1: GenEnv (Princeton, arXiv 2512.19682)
- **标题**: Difficulty-Aligned Co-Evolution Between LLM Agents and Environment Simulators
- **核心**: α-课程奖励 Renv(p̂)=exp(−β(p̂−α)²)，p=0.5最大化梯度
- **结果**: API-Bank +17.5%, ALFWorld +40.3%, 比Gemini方案少3.3x数据

### 论文2: DreamGym (Meta Research, ICLR 2026, arXiv 2511.03773)
- **标题**: Scaling Agent Learning via Experience Synthesis
- **核心**: 推理型经验模型 + 经验回放缓冲区 + 自适应课程学习
- **结果**: WebArena超基准30%+，关闭课程后各降6pp

### 论文3: AutoForge (arXiv 2512.22857)
- **标题**: Automated Environment Synthesis for Agentic Reinforcement Learning
- **核心**: 工具文档→DB→Python→依赖图→DAG蓝图，ERPO=GRPO+4项扩展
- **结果**: 10虚拟环境含1078高难度任务

### 论文4: AgentFrontier (ICLR 2026, arXiv 2510.24695)
- **标题**: Expanding the Capability Frontier of LLM Agents with ZPD-Guided Data Synthesis
- **核心**: ZPD双角色(LKP+MKO)，三阶段合成(种子→精炼→边界筛选)
- **结果**: 30B-A3B达HLE 28.6%，149赞/85收藏

## 十、核心结论

论文提供的**思想框架**有真实价值——特别是"在边界处密集训练"和"合成弱类数据"两个方向。但**具体方法不可直接照搬**。最大的差距在于反馈信号：RL Agent有环境奖励，而NLU分类器没有。

**建立反馈闭环是所有后续优化的前提，这一点比任何论文方法都重要。**

建议投资优先级：
1. **反馈机制 + 路由统计持久化**（基础设施，非论文方法）
2. **P4 工具文档测试**（低风险，AutoForge启发）
3. **P0 OOD激活 + 阈值微调**（利用已有代码）
4. **P2 弱类合成**（DreamGym/GenEnv启发，需标签审计后）
5. **P1 ZPD闭环**（AgentFrontier启发，需反馈机制就位后）
