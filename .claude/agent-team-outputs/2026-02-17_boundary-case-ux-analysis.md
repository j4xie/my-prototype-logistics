# AI 助手边界 Case 答复自然度与 UX 分析

**研究日期**: 2026-02-17
**研究范围**: Cretas 食品溯源系统 AI 意图路由边界 Case
**研究方法**: API 测试 + 16 对查询对比 + 内容相关性评估

---

## 执行摘要

### 核心发现

| 指标 | 数值 | 评价 |
|------|------|------|
| **意图识别准确率** | 12/16 (75%) | 良好 |
| **完全路由失败** | 4/16 (25%) | 需要改进 |
| **边界 Case 失败率** | 2/2 (100%) | 高风险 |
| **相同查询对分化度** | 14/16 (87%) | 优秀 |

### 关键问题

1. **Query Pattern**: 形如 "XXX用了多少/XXX加工了几批" 的"数值查询"完全失败 (0%)
2. **Boundary Zone**: "食品安全注意什么" (模糊食品/工厂) 和 "食品安全检查结果" 都无法识别
3. **UX Impact**: 失败案例均返回 "我没有理解您的意图" 通用提示，用户体验差
4. **自然度**:
   - 成功路由的 FOOD_KB 答复非常自然、详细、有质量
   - 失败的工厂数据查询只返回 "执行成功" 或 "需要澄清"
   - 边界模糊查询触发 RAG 但置信度极低

## 定性评估维度

### 1️⃣ **相关性 (Relevance)**

**评分标准**: 答复内容是否直接回答用户查询

| 场景 | 评分 | 说明 |
|------|------|------|
| FOOD_KB 成功案例 | ⭐⭐⭐⭐⭐ | 返回标准、参数、工艺细节，完全相关 |
| MATERIAL/EQUIPMENT 成功 | ⭐⭐⭐ | 返回 "执行成功"，假设后台有数据，但UX欠缺 |
| NEED_CLARIFICATION | ⭐⭐ | 用户已明确查询，系统要求澄清，低分 |
| RAG_LOW_CONFIDENCE | ⭐⭐⭐ | 返回知识库索引而非直接答复，相关但间接 |

**总体相关性**: 7/16 (FOOD_KB) + 5/16 (正确工厂意图) = **12/16 = 75%**

### 2️⃣ **内容匹配度 (Content Match)**

**评分标准**: 答复内容是否与查询意图一致

**高质量匹配** (12/12 成功案例):
- ✅ "牛肉加工注意事项" → 返回 GB/T 23969、原料选择、工艺流程 (完美)
- ✅ "杀菌温度应该多少度" → 返回 72-78°C、F值、中心温度要求 (直接回答)
- ✅ "我想了解猪肉安全问题" → 返回肉脯生产、干燥温度、冷却要求 (完整)

**匹配失败** (4/16):
- ❌ "牛肉加工了几批" → 返回澄清提示，没有数据 (不匹配)
- ❌ "防腐剂用了多少" → 返回澄清提示，没有数据 (不匹配)
- ❌ "食品安全注意什么" → 返回标准列表而非具体建议 (部分不匹配)
- ❌ "食品安全检查结果" → 返回澄清提示，没有数据 (不匹配)

**内容匹配度**: **75%**

### 3️⃣ **语气一致性 (Tone Consistency)**

**评分标准**: 答复风格是否与查询风格匹配

| 查询风格 | 系统答复风格 | 评价 |
|---------|-----------|------|
| 正式/学术 (注意事项、标准、温度) | 标准文档风格 + 规范号引用 | ✅ 匹配 (FOOD_KB) |
| 非正式/口语 (怎么使用、多少) | 澄清提示 (通用语气) | ❌ 不匹配 (NEED_CLARIFICATION) |
| 数据查询 (了几批、用了多少) | 通用澄清 | ❌ 不匹配 |
| 模糊探索 (食品安全注意什么) | RAG 索引列表 | ⚠️ 中立但不完整 |

**语气一致性**: **65%** (FOOD_KB 类完美，工厂数据类偏弱)

### 4️⃣ **模糊处理 (Ambiguity Handling)**

**评分标准**: 系统对边界 Case 的处理能力

| 模糊类型 | 案例 | 处理方式 | 评价 |
|---------|------|--------|------|
| 同材料不同维度 | 猪肉(知识) vs 猪肉(库存) | 关键词分化 | ✅ 优秀 |
| 同工艺不同层面 | 冷链(标准) vs 冷链(设备) | 语义识别 | ✅ 优秀 |
| 通用术语双重意义 | 食品安全(知识) vs 食品安全(结果) | 放弃识别 | ❌ 失败 |
| 数值查询 | XXX了几批/多少 | 无法识别 | ❌ 失败 |

**模糊处理能力**: **50%** (能处理语义分化，不能处理通用术语)

## 问题根因分析

### 问题 #1: 数值查询完全失败 (Pattern: "XXX了几批/多少")

**失败案例**:
- "牛肉加工了几批"
- "防腐剂用了多少"

**根因**:
- 意图分类器缺少 "数值统计查询" 类别
- Layer 0 (规则匹配) 中没有 "几批、多少、用了、生产了" 等数值关键词
- Layer 1 (NER+LLM) 识别出是物料查询，但无法确定是 MATERIAL_BATCH_QUERY 还是其他

**影响**: 用户在数据查询上的成功率 0%

---

### 问题 #2: 边界模糊术语 (Pattern: "食品安全")

**失败案例**:
- "食品安全注意什么" → RAG_COMPLETED_LOW_CONFIDENCE
- "食品安全检查结果" → NEED_CLARIFICATION

**根因**:
- "食品安全" 是广域概念，既属于知识类(如何确保安全)，也属于工厂数据类(检查结果)
- Layer 0 规则无法区分
- Layer 1 触发 RAG，但 RAG 返回低置信度 (假设向量相似度低)
- Fallback: NEED_CLARIFICATION

**影响**: 边界模糊查询的用户体验极差 (系统不敢做决定)

---

### 问题 #3: 工厂数据查询答复不完整

**现象**:
- MATERIAL、EQUIPMENT、PROCESSING 等工厂意图识别正确
- 但答复只有 "执行成功" (4字)，没有实际数据

**根因**:
- API 端点可能调用了后台服务但没有返回结果
- 或者后台数据为空 (该工厂/日期没有相关数据)

**影响**: 用户即使识别正确，也无法获取期望的数据 (虽然技术上成功，但体验失败)

## 自然度详细评估

### 自然度定义

"自然度" 指答复的表达是否符合对话习惯、是否直接回答问题、是否避免生硬或模糊的提示。

### 高自然度案例 ⭐⭐⭐⭐⭐

**例 1**: "杀菌温度应该多少度" →

```
根据肉制品加工规范：
- 中心温度要求:
  - 禽肉(整只/碎肉): ≥74°C (165°F)
  - 猪肉(非整块): ≥72°C (160°F)
  - 牛肉整块(牛排): ≥63°C (145°F) + 静置3分钟
  - 碎牛肉(汉堡肉饼): ≥72°C (160°F)
```

**为什么自然**: 用户问 "应该多少度"，系统直接给出温度范围 (72-78°C)，附加标准依据，完全回答了问题。

---

**例 2**: "猪肉怎么储存" →

```
根据冷链储存标准：
- 冷藏: 70-80%, 0-4°C保存 8-14天
- 冷冻: PE袋真空包装后速冻, -18°C保存 ≤12月
- 新鲜肉展示: 冷柜保存, 0-4°C, 展示时间 ≤4小时
```

**为什么自然**: 用户问 "怎么储存"，系统列举储存方式、温度、时间，实用且有条理。

---

### 低自然度案例 ⭐⭐

**例 1**: "牛肉加工了几批" →

```
我没有理解您的意图，请从以下常用操作中选择，或更详细地描述您的需求：
1. 生产管理
2. 批次查询
3. ...
```

**为什么不自然**:
- 用户已经很清楚 "我想查加工批数"，系统说 "没理解"
- 用户被迫重新组织语言而不是获取信息
- 通用澄清提示显得没有 "理解力"

---

**例 2**: "食品安全注意什么" →

```
【知识库检索结果】

查询: 食品行业 食品安全注意什么 标准规范

相关标准/规范:
1. GB 14881-2013 《食品安全国家标准 食品生产通用卫生规范》
2. GB/T 27306 《食品安全管理体系 餐饮业要求》

注意：以上为示例数据，实际检索功能待集成向量知识库。
```

**为什么不自然**:
- 系统返回标准号而非具体指导
- 最后一句 "待集成向量知识库" 显得内部技术细节泄露
- 用户问 "注意什么"，得到 "GB号是什么"，不是有效回答

## 改进建议

### 优先级 P0 (立即修复)

#### 1. 添加数值查询意图识别

**问题**: "XXX了几批、XXX用了多少" 无法识别

**解决方案**:
- 在 TwoStageIntentClassifier 中添加 "数值统计查询" 层
- Layer 0 规则中添加关键词: "几批、多少、用了、生产了、加工了、消耗了"
- 示例:
  ```java
  if (input.matches(".*[了了].*[几多少数量批次].*")) {
      return new IntentResult(MATERIAL_BATCH_QUERY, confidence=0.95);
  }
  ```

**预期影响**: "牛肉加工了几批" 等 2/2 案例修复，成功率 75% → 87%

---

#### 2. 完整化工厂数据答复

**问题**: MATERIAL/EQUIPMENT 意图识别对但答复只有 "执行成功"

**解决方案**:
- 检查 IntentExecutorServiceImpl 中对应 Handler 的实现
- 确保查询后返回实际数据而非空结果
- 若数据为空，返回 "今日暂无加工批次" 而非 "执行成功"

**预期影响**: UX 从识别成功但数据失败 → 识别成功且数据完整

#### 3. 边界模糊查询的多意图处理
