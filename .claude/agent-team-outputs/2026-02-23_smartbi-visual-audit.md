## 四、LLM 深度加工需求

基于截图分析，以下区域需要增加 LLM 处理深度:

### 4.1 列名语义映射 (P1 优先)

**现状**: KPI 卡片直接使用原始列名 "实际收入②"
**需求**: LLM 根据上下文推断列的业务含义
**方案**: 在 `getSmartKPIs()` 中增加 LLM call，输入列名 + 同 sheet 其他列名 + 数据样本，输出业务语义名称

```
输入: "实际收入②", context: ["实际收入③", "预算收入②", sheet: "收入及净利简表"]
输出: "Q2 实际收入" 或 "第二季度实际收入"
```

### 4.2 图表标题智能化 (P1)

**现状**: `generateChartTitle()` 纯规则拼接，会产生 "项目、项目 双指标对比" 等不合理标题
**需求**: LLM 生成自然语言标题
**方案**: 对 `buildChartPlan()` 输出的 plans，用 LLM 批量生成描述性标题

### 4.3 图表选型 LLM 辅助 (P2)

**现状**: 260 行利润表的子表全部用相同图表组合 (柱状+饼图+双指标+增减)
**需求**: LLM 根据子表语义选择差异化图表 — 收入子表用趋势线，费用子表用占比饼图，利润子表用瀑布图
**方案**: 在 `buildChartPlan()` 的 Phase 1 前增加 LLM 分类步骤

### 4.4 AI 分析个性化 (P3)

**现状**: 所有 sheet 的 AI 分析结构相同 (风险提示 + 趋势 + 建议)
**需求**: 根据数据特征生成差异化分析 — 销售明细强调客户分析，利润表强调预算执行率

3. **Y 轴单位自动缩放**: 已有 `enhanceChartOption()` 处理 `(万)`/`(亿)` 的逻辑，但未覆盖无括号场景 — 当 max > 100000 时自动加 "万" 缩放
